
# `next/head` (Pages Router)

`next/head` is a built-in component in Next.js (primarily for the Pages Router) that allows you to modify the `<head>` of the page. This is crucial for setting metadata, title, meta tags, and linking external resources like stylesheets or fonts, which are important for SEO and social media sharing.

### When to use it
-   In Pages Router applications, to manage page-specific `<head>` content.
-   To set dynamic titles, meta descriptions, Open Graph tags, etc.
-   To include external CSS or fonts that are specific to a page.

### Syntax

```tsx
import Head from 'next/head';

function MyPage() {
  return (
    <>
      <Head>
        <title>My Page Title</title>
        <meta name="description" content="Description of my page" />
        {/* Other head elements */}
      </Head>
      {/* Page content */}
    </>
  );
}
```

### Props
`next/head` doesn't have specific props; instead, you place standard HTML head elements (`<title>`, `<meta>`, `<link>`, `<script>`) as its children.

### Example

This example demonstrates how to use `next/head` to set a dynamic page title and meta description.

```tsx
// pages/nextjs-apis/head/index.tsx
'use client';

import Head from 'next/head';
import { useState } from 'react';

export default function HeadExample() {
  const [pageTitle, setPageTitle] = useState('Default Title');

  return (
    <>
      <Head>
        <title>{pageTitle}</title>
        <meta name="description" content={`This is a page about ${pageTitle.toLowerCase()}.`} />
        <meta property="og:title" content={pageTitle} />
        <meta property="og:description" content={`Learn more about ${pageTitle.toLowerCase()}.`} />
        <link rel="icon" href="/favicon.ico" />
      </Head>

      <div>
        <h1>Next.js Head Example</h1>
        <p>Current Page Title: {pageTitle}</p>
        <button onClick={() => setPageTitle('Dynamic Page')}>Set Dynamic Title</button>
        <button onClick={() => setPageTitle('Another Title')}>Set Another Title</button>
        <p>Check your browser tab title and view page source to see the changes.</p>
      </div>
    </>
  );
}
```

### Note on App Router

In the App Router, metadata is handled differently and more powerfully using a file-based metadata API. You define metadata by exporting `metadata` objects or `generateMetadata` functions in `layout.tsx` or `page.tsx` files. This replaces the need for `next/head` in most App Router scenarios.

Example for App Router (`layout.tsx` or `page.tsx`):

```typescript
// app/layout.tsx or app/page.tsx
export const metadata = {
  title: 'My App',
  description: 'Generated by Next.js',
};

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

### Key Points
-   `next/head` is primarily for Pages Router.
-   For App Router, use the file-based metadata API for SEO and social media tags.
-   It helps manage the `<head>` section of your HTML document, which is critical for search engine optimization (SEO) and how your content appears when shared on social media.
